# Contributing

Thanks for helping. This project is open source and community-friendly, but it is also founder-led so it doesn’t drift.

## Ways to contribute (no coding required)
- Report bugs (screenshots + steps)
- Suggest improvements (describe the user goal)
- Improve docs (clarify a confusing section)
- Share sample datasets (historical anchors, timelines)

## Before you open an issue
Include:
1. What you expected
2. What happened
3. Steps to reproduce
4. Browser + OS
5. Screenshot/video if possible

## Pull requests (PRs)
### Basic rules
- Keep PRs small and focused (one change per PR)
- Explain *why* the change matters
- Include before/after screenshots if UI changes
- Avoid drive-by refactors

### Process
1. Fork the repo
2. Create a feature branch
3. Make your change
4. Open a PR

### Style & quality
- Prefer clarity over cleverness
- Comment “why,” not “what”
- If you add a feature, update docs

## Decision process
The maintainer reviews PRs and decides what merges. If you want a bigger change, open an issue first so we can align.
